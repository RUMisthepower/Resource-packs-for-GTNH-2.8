# 📦 Инструкция по созданию релиза

Этот документ описывает процесс создания релиза текстурпака с автоматической генерацией ZIP-архива.

---

## 🎯 Как это работает

При создании нового релиза на GitHub:
1. ✅ GitHub Actions автоматически собирает ZIP-архив
2. ✅ Архив прикрепляется к релизу
3. ✅ Пользователи могут скачать готовый пак

---

## 📋 Шаг за шагом

### Шаг 1: Подготовьте изменения

Убедитесь, что все изменения закоммичены и отправлены в репозиторий:

```bash
git status
git add .
git commit -m "feat: описание изменений"
git push origin main
```

---

### Шаг 2: Создайте тег версии

Тег должен иметь формат `vX.Y.Z` (например, `v1.15`):

```bash
# Создайте тег
git tag v1.15

# Отправьте тег на GitHub (это запустит авто-сборку!)
git push origin v1.15
```

> ⚠️ **Важно**: После отправки тега (`git push origin v1.15`) GitHub Actions **автоматически начнёт сборку ZIP-архива**.

---

### Шаг 3: Дождитесь сборки

После отправки тега:
1. ⏳ GitHub Actions начнёт сборку (обычно 1-2 минуты)
2. 🟡 Статус сборки отображается на странице релиза
3. 🟢 Когда сборка завершится, ZIP-архив появится в разделе **Assets**

---

### Шаг 4: Проверьте релиз

Убедитесь, что:
- ✅ ZIP-архив появился в разделе **Assets**
- ✅ Архив содержит все необходимые файлы
- ✅ Текстуры корректно работают в игре

---

## 🔧 Что входит в ZIP-архив

Автоматически включаются:
- ✅ Все текстуры и ресурсы
- ✅ `pack.mcmeta` (метаданные пака)
- ✅ `LICENSE` (лицензия)

**Не включаются:**
- ❌ Файлы `.git/` (служебные)
- ❌ `.github/` (конфигурация CI/CD)
- ❌ `.md` файлы (документация — она есть в репозитории)
- ❌ `docs/` (документация)

## ❓ Вопросы

### Что делать, если сборка не началась?
1. Проверьте, что тег имеет формат `vX.Y.Z`
2. Убедитесь, что файл `.github/workflows/release.yml` существует
3. Проверьте вкладку **Actions** на странице репозитория

### Как посмотреть логи сборки?
1. Перейдите на https://github.com/RUMisthepower/All-in-one/actions
2. Выберите запуск с названием релиза
3. Нажмите на шаг **build** для просмотра логов

### Что делать, если сборка завершилась ошибкой?
1. Проверьте логи сборки
2. Убедитесь, что все файлы на месте
3. Попробуйте создать релиз заново
