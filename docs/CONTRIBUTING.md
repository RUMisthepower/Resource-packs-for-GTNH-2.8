## 🚀 С чего начать

### 1. Форк репозитория

Нажмите кнопку **Fork** в правом верхнем углу страницы репозитория на GitHub.

### 2. Клонируйте репозиторий

```bash
git clone https://github.com/<ваш-username>/All-in-one.git
cd All-in-one
```

### 3. Настройте upstream

```bash
git remote add upstream https://github.com/RUMisthepower/All-in-one.git
```

### 4. Создайте ветку

```bash
git checkout -b feat/ваша-функция
```

## 🎨 Требования к текстурам

### Технические требования

| Параметр | Значение |
|----------|----------|
| **Размер** | 32×32 пикселя |
| **Формат** | PNG |
| **Цветовое пространство** | sRGB |
| **Прозрачность** | Поддерживается (alpha-канал) |

### Требования к качеству

- ✅ Текстуры должны соответствовать общему стилю пака
- ✅ Избегайте артефактов сжатия
- ✅ Сохраняйте читаемость деталей на расстоянии
- ✅ Проверяйте текстуры в игре перед отправкой


## 📝 Процесс внесения изменений

### 1. Внесите изменения

Добавьте новые текстуры или измените существующие в соответствующих папках.

### 2. Проверьте изменения

Запустите Minecraft и убедитесь, что текстуры корректно отображаются.

### 3. Закоммитьте изменения

```bash
git add .
git commit -m "feat: добавил текстуры для AE2"
```

### 4. Отправьте в репозиторий

```bash
git push origin feat/ваша-функция
```

### 5. Создайте Pull Request

1. Перейдите на страницу вашего форка
2. Нажмите **Compare & pull request**
3. Заполните описание изменений
4. Укажите связанные Issue (если есть)
5. Нажмите **Create pull request**

## ✍️ Правила коммитов

Прошу вас следовать соглашению [Conventional Commits](https://www.conventionalcommits.org/ru/):

### Формат

```
<тип>: <описание>
```

### Типы коммитов

| Тип | Описание |
|-----|----------|
| `feat` | Новая функция |
| `fix` | Исправление ошибки |
| `docs` | Изменения в документации |
| `style` | Изменения в оформлении (пробелы, форматирование) |
| `refactor` | Рефакторинг кода |
| `perf` | Улучшение производительности |
| `chore` | Изменения в сборке, зависимостях и т.д. |

### Примеры

```bash
feat: добавил текстуры для AE2
fix: исправил артефакты на текстуре булыжника
docs: обновил README.md
refactor: оптимизировал структуру папок
```

## 🔍 Code Review

### Что проверяется

- ✅ Соответствие техническим требованиям (32×32, PNG)
- ✅ Качество текстур
- ✅ Совместимость с другими текстурами
- ✅ Правильность именования файлов
- ✅ Описание изменений в PR

### Время проверки

Обычно Pull Request проверяется в течение **3-5 дней**.

### Внесение правок

Если ревьювер запросил изменения:

1. Внесите правки в своей ветке
2. Закоммитьте: `git commit -m "fix: исправил замечания"`
3. Отправьте: `git push origin feat/ваша-функция`

PR обновится автоматически.

## 📜 Лицензирование

Отправляя свои работы в этот проект, вы соглашаетесь с тем, что:

- Ваши текстуры будут распространяться под лицензией **CC BY-NC-SA 4.0**
- Вы являетесь автором отправляемых текстур **ИЛИ** имеете право на их распространение
- Ваше имя будет добавлено в список [контрибьюторов](./CONTRIBUTORS.md)

> ⚠️ **Важно**: Не отправляйте текстуры, украденные у других авторов или нарушающие чьи-либо авторские права.
